Product Identification

Client Quality Control in Factory (Same with Sub-cont Management)
Sector Quality Production
My Role Entire product conception and design
Project Time 3 months

Disclaimer : Based on the agreement I will not expose the factory name.

This case has the same client as my last project about sub-cont management. This time, I make countermeasures for a quality problem issue that is causing Customer Claims for the Factory.

This problem happens at the station after the product is assembled before it gets packaged. There is a station called “OK Packing” which is responsible for judging the product which is come from a lot of production lines match the product specification. The problem occurs because the is a miss judgment or wrong judgment on that station. Why is it happening? let’s dive deeper.

Learn The Case

Look at this process illustration

This station has a super big job but the current process needs to have proper technical execution. My client also mentioned that the procedure standard is need to be reviewed again. The big problem is, there is no strong proof that the process is well executed, why?, Because determining the product is correct only depends on user memory and experience. To handle almost 800 product variants is so risky. Can you imagine that? and also that person is working his job in shift time not only in the day time.

And now for this issue, my client proposes a specification that needs to be fulfilled, it is “we have to change the inspection process”. Ok, now what should we do next? fo course, get some data thru interviews and surveys.

Wireframing, Prototyping & Usability-Testing

The sequential work will be divided into 2 branches which are about :

  1. Normal Flow (for inspection)
  2. Teaching Flow (for teaching the machine learning model)

These 2 divided areas will be separated by who is using this app now. We talk about the regulation and the Standard of Operation of this new system. It is so important to register the user level to prevent big problems such as wring teaching for the model which will cause a really big problem for the machine learning technology.

Mood Board plus Design System

I will refer to my previous mood board because it has the same client. Applying the factory culture will be the best option for smoothing the system transition. and make it consistent in the digital system.

Iterating

After testing the prototype, there are some great suggestions from the user. Redesign the wireframe and test it again to the user for double check and make it more and more perfect. There is no to many iterations on this case because earlier test which is conducted help me to reduce the possibility of design changes.

What Next?

This is when I ask the system integrator or the software developer to make the design live. This area will be handled by the Factory IT section. But on my side there are some points that should be considered:

  1. Integrating the system with an old ERP system
  2. Keep my KPI on maintaining the unsuccessful workflow
  3. Expanding the system for another building in the factory

Conclusion

In this case, a really big change is should be done. A big gap between user computing ability and the solution that should be implemented is a big and new experience for me. Bridging the gap is where the UX designer should be right? Keeping always put my user right at the front and center to prevent some bias in my design solution.

Learnings

In this case, I learned that a strong bridge does not only talk about the bridge material but there are a lot of considerations, surveys, planning, and reviews to make it work well and be safe for the users. Also with this case, I hope my bridge will deliver my user to their destination smoothly while enjoying the view hehe. #
